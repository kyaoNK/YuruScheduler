### Phase 7: 視覚的警告機能 🚨

#### 7.1 締切の色分けロジック

```typescript
function getCardColor(dueDate: Date): string {
  const today = new Date();
  const daysUntilDue = Math.floor((dueDate.getTime() - today.getTime()) / (1000 * 60 * 60 * 24));

  if (daysUntilDue < 0) return 'bg-red-800'; // 期限超過: 濃い赤
  if (daysUntilDue <= 3) return 'bg-red-500'; // 0〜3日: 赤
  if (daysUntilDue <= 7) return 'bg-yellow-400'; // 3〜7日: 黄色
  return 'bg-gray-100'; // 7日以上: 通常色
}
```

## 開発時のポイント

### データフロー

1. ユーザーが手動カードを作成 → 公開日から自動カードを生成
2. 自動カードを編集 → 手動カードに変換
3. 公開日を変更 → 自動カードのみ再計算（手動カードは変更なし）

### パフォーマンス

- カードの数が増えても快適に動作するように、リスト表示は仮想スクロール（react-window等）を検討
- 日付計算のキャッシュ化

### UI/UX

- カード作成は素早くできるようにショートカット対応
- ドラッグ&ドロップで直感的な操作を提供（任意）
